# 申论 Agent 接口设计

## 设计目标

接口层负责承接前端请求，并将申论训练任务分发给 Agent 编排层与模型适配层，确保不同模型和配置能够统一接入。

## 核心接口

### 1. 题目分析

- **接口**：`POST /api/analyze`
- **用途**：分析申论题目，识别题型、要求、作答重点
- **输入**：题目文本、可选背景材料、用户配置 ID
- **输出**：题型、关键词、作答要求、建议切入点

### 2. 提纲生成

- **接口**：`POST /api/outline`
- **用途**：基于题目生成参考作答提纲
- **输入**：题目文本、分析结果、配置参数
- **输出**：总分结构、分段要点、展开建议

### 3. 答案批改

- **接口**：`POST /api/review`
- **用途**：对用户答案进行批改、评分与点评
- **输入**：题目、用户答案、参考要求、配置参数
- **输出**：评分、优点、问题、改进建议

### 4. 配置管理

- **接口**：`POST /api/config`
- **用途**：保存用户自己的 AI 配置
- **输入**：模型名称、API Key、温度、系统提示词等
- **输出**：配置 ID 与保存结果

### 5. 练习记录

- **接口**：`GET /api/history`
- **用途**：查询用户练习与批改记录
- **输出**：历史题目、答案、评分、复盘摘要

## 统一响应格式

建议所有接口返回统一结构：

- `success`：是否成功
- `message`：提示信息
- `data`：核心结果
- `error`：错误信息（失败时返回）

## 设计原则

- 接口尽量保持简洁
- 所有 AI 参数均可配置
- 同一接口可兼容不同模型
- 后续支持流式输出与多轮交互
