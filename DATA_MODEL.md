# 申论 Agent 数据模型

## 目标

数据模型用于保存题目、答案、配置、批改结果和训练记录，支撑申论训练闭环。

## 核心实体

### 1. 用户配置 `UserConfig`

- 配置 ID
- 模型提供方
- 模型名称
- API Key
- Base URL
- 温度
- 输出长度限制
- 系统提示词

### 2. 题目记录 `QuestionRecord`

- 题目 ID
- 题目文本
- 题型
- 题目标签
- 输入时间

### 3. 答案记录 `AnswerRecord`

- 答案 ID
- 题目 ID
- 用户答案文本
- 提交时间
- 关联配置 ID

### 4. 批改结果 `ReviewResult`

- 批改 ID
- 答案 ID
- 评分
- 优点
- 问题
- 修改建议
- 复盘结论

### 5. 练习历史 `PracticeHistory`

- 历史 ID
- 题目 ID
- 答案 ID
- 批改结果 ID
- 完成时间
- 是否收藏

## 数据关系

- 一个用户配置可对应多次练习
- 一个题目可对应多个答案版本
- 一个答案只会对应一个批改结果
- 一个练习历史记录关联题目、答案和批改结果

## 存储建议

- 本地阶段：MySQL（可使用本地 Docker 或直接安装）
- 团队/部署阶段：MySQL 主库 + 备份策略
- 配置与 Prompt 模板：可单独保存为可导入导出文件
