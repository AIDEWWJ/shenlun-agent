# 申论 Agent MVP 开发计划

## MVP 目标

先做出一个能用、能验证价值的最小版本：输入申论题目与答案，系统可分析题目、生成提纲、完成基础批改，并支持用户自定义模型配置。

## 第一版必须实现的能力

1. 用户配置 AI
   - 支持填写 API Key
   - 支持选择模型
   - 支持配置温度、系统提示词

2. 题目分析
   - 输入题目文本
   - 输出题型、关键词、作答要求

3. 提纲生成
   - 根据题目生成参考提纲
   - 输出分段结构和要点

4. 答案批改
   - 输入用户答案
   - 输出评分、优点、问题与改进建议

5. 练习记录
   - 保存题目、答案和批改结果
   - 支持查看历史记录

## 开发顺序建议

### 阶段一：项目骨架

- 创建前后端基础目录
- 初始化 Vue 3 前端
- 初始化 FastAPI 后端
- 建立 MySQL 连接

### 阶段二：核心链路

- 接入 LangChain
- 打通题目分析接口
- 打通提纲生成接口
- 打通答案批改接口

### 阶段三：用户配置与记录

- 做模型配置页
- 做练习记录页
- 保存与读取历史数据

### 阶段四：体验优化

- 加入流式输出
- 优化提示词模板
- 增加错题本与收藏功能

## 风险提示

- LangChain 适合快速落地，但复杂状态流后续可能需要 LangGraph
- AI 输出稳定性需要通过提示词和结构化输出约束
- 数据库设计要尽量避免后期大改表结构
