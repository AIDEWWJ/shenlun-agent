# 申论 Agent 角色设计

## 一、系统用户角色

### 1. 学员 / 普通用户

面向核心使用者。

**权限：**

- 创建和管理自己的 AI 配置
- 输入题目并生成提纲
- 提交答案并查看批改结果
- 查看历史练习与收藏记录
- 编辑个人提示词模板

### 2. 管理员

面向系统维护者。

**权限：**

- 管理用户账号
- 管理全局模型配置和系统默认 AI 配置
- 管理全局 Prompt 模板
- 查看系统运行状态
- 管理题库与基础数据

### 3. 访客

未登录或仅浏览状态的用户。

**权限：**

- 只查看公开介绍页面
- 不可保存数据
- 不可使用完整 Agent 能力

## 二、Agent 职能角色

为了让系统输出更像“辅导老师”，建议把 AI 内部拆成多个职能角色：

### 1. 题目分析师

- 识别题型
- 提取题目关键词
- 归纳作答要求

### 2. 提纲生成器

- 根据题目生成结构化提纲
- 提供总分总或分段式写法建议

### 3. 批改官

- 评估答案完整度
- 判断逻辑与表达问题
- 给出分数和修改建议

### 4. 复盘教练

- 总结本次练习得失
- 给出下一步训练建议
- 生成错题复盘结论

## 三、推荐权限边界

- 普通用户只操作自己的数据
- 管理员可以维护全局配置和题库
- Agent 职能角色只负责“推理与生成”，不直接持久化业务数据

## 四、实现建议

- 系统权限用 RBAC
- Agent 职能通过 LangChain 工具链或提示词拆分实现
- 角色与能力不要混在一张表里，保持清晰
