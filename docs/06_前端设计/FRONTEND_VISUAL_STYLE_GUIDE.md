# 申论 Agent 前端视觉风格规范

> 适用范围：所有前端页面、组件、空状态、弹窗、表单、导航与信息面板。
>
> 约束要求：每次新增或重构前端页面前，必须先阅读本规范，再进行视觉设计与组件选择。

## 1. 风格定位

本项目的前端视觉应服务于“申论刷题、写作训练、AI 批改与结构化思考”的使用场景，整体气质应当：

- 专业
- 安静
- 可信
- 克制
- 耐看
- 适合长时间阅读与训练

它更像一个高质量的写作训练工作台、知识工具和 AI 陪练平台，而不是炫酷的 AI 营销站或娱乐化产品。

## 2. 设计关键词

核心关键词：

- 申论训练
- 写作提升
- 结构化思考
- AI 批改
- 学习工作台
- 现代知识工具
- 安静专注

## 3. 视觉气质要求

### 应该呈现的感觉

- 像一本排版精良的工作手册
- 像一间安静、清晰、效率优先的训练工作室
- 像严肃但不压迫的知识型产品
- 像可长期使用的专业写作平台

### 不应该呈现的感觉

- 霓虹科技风
- 强发光、赛博朋克
- 卡通插画风
- 儿童教育风
- 过度玻璃拟态
- 娱乐化、游戏化视觉
- 过重的动效和花哨装饰

## 4. 色彩体系

### 主色方向

- 浅米白：用于页面背景、主容器底色、空白区域
- 暖灰：用于次级背景、辅助卡片、分隔区域
- 墨蓝灰：用于正文、标题、关键文本
- 棕铜色：用于品牌主强调色、主按钮、选中态、重要提示
- 低饱和蓝灰：用于辅助标签、次级状态、轻量信息提示

### 推荐配色语义

- 背景：浅米白 / 暖米灰
- 主文字：墨蓝灰
- 次文字：中性暖灰
- 品牌强调：棕铜色
- 辅助强调：低饱和蓝灰
- 分割线：低对比暖灰边框

### 颜色使用原则

- 强调色只用于关键动作，不要大面积铺满页面
- 背景色要柔和，不要抢内容
- 文本对比度必须足够，保证长时间阅读舒适
- 避免高饱和蓝、紫、青等强科技感颜色

## 5. 字体与排版

### 字体气质

- 中文正文偏现代、清晰、耐读
- 标题可带一点书卷感或轻微衬线感
- 整体不要过于装饰化

### 排版原则

- 标题简洁，避免口号化长句
- 内容层级清晰，H1、H2、H3 区分明确
- 行高充足，适合长时间阅读
- 段落不要过长，尽量分段表达
- 表单标签、帮助说明、错误提示要清晰可辨

## 6. 布局与结构

### 页面结构

- 页面版心清晰，宽度受控
- 留白充足，避免拥挤
- 采用卡片化布局表达功能分区
- 重点内容优先可视化，弱化无关说明
- 复杂页面优先使用左右分栏或主次分区

### 结构原则

- 主任务必须第一眼可见
- 页面信息层级必须从“目标 → 操作 → 反馈”展开
- 不要把功能堆在单一大组件里
- 训练、批改、配置、管理类页面要明显区分

## 7. 卡片、边框、阴影、圆角

### 卡片风格

- 使用轻边框、轻阴影、柔和背景
- 圆角偏大，但不要夸张到娱乐化
- 卡片要有明确边界，但不能厚重

### 推荐感觉

- 安静
- 轻盈
- 纸张感
- 工具感

### 避免

- 重投影
- 大面积毛玻璃
- 夸张高亮
- 强烈发光边缘

## 8. 按钮与交互

### 主按钮

- 仅用于最重要动作：登录、提交、保存、生成、确认
- 颜色应稳定、克制、可信
- 不宜过高亮或过“营销”

### 次按钮

- 用于切换、取消、辅助操作
- 保持低干扰

### 交互要求

- 状态切换清晰
- loading、禁用、成功、失败必须可感知
- 错误提示应尽量具体，不使用空泛文案

## 9. 表单与阅读体验

申论训练平台会有较多长时间表单与阅读行为，因此应特别注意：

- 输入框要清晰、易点按
- 标签与输入内容关系明确
- 帮助文案简短、自然、非技术化
- 错误提示尽量显示在字段附近
- 表单不要过密
- 保证移动端首屏即可看到关键操作

## 10. 文案风格

### 推荐文案语气

- 自然
- 克制
- 面向用户
- 少技术术语

### 不推荐文案

- token、Bearer、请求头、调试接口等工程术语
- 过度宣传式口号
- 过于热闹或轻浮的表达

### 文案表达原则

- 说清楚“用户能做什么”
- 说清楚“系统会给什么帮助”
- 少说实现原理，多说使用结果

## 11. 页面风格总要求

每次设计页面时，都应满足以下判断：

1. 是否一眼看出当前页面的核心任务？
2. 是否足够安静、耐看、适合长时间使用？
3. 是否避免了炫酷 AI 营销站的感觉？
4. 是否让阅读、写作、批改、配置等行为更高效？
5. 是否符合申论训练工作台的专业气质？

如果答案是否定的，应继续收敛视觉并减少干扰元素。

## 12. 前端设计落地清单

新增或修改页面时，请先检查：

- [ ] 页面主任务是否足够明确
- [ ] 是否使用浅米白、暖灰、墨蓝灰主色体系
- [ ] 棕铜色是否只用于关键强调
- [ ] 卡片、边框、阴影是否足够轻
- [ ] 圆角是否统一且克制
- [ ] 是否存在无关装饰或过强发光
- [ ] 是否符合申论训练与写作工作台气质
- [ ] 是否已考虑长时间刷题的阅读舒适性
- [ ] 是否已考虑移动端单栏适配
- [ ] 是否已避免技术术语直接暴露给普通用户

## 13. 默认设计结论

本项目的默认视觉方向应始终保持：

**专业、安静、可信、克制、书卷感 + 现代工具感。**
