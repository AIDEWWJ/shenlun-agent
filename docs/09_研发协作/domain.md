# 领域文档

工程类 skills 在探索代码库时如何读取本项目的领域文档。

## 探索前先读这些文件

- **`CONTEXT.md`** — 在项目根目录，或
- **`CONTEXT-MAP.md`** — 如果存在，它指向各子目录各自的 `CONTEXT.md`，按需读取
- **`docs/adr/`** — 读取与当前工作相关的架构决策记录（ADR）

如果这些文件**不存在，跳过即可**。不要提示缺失，也不要建议创建。`/grill-with-docs` skill 会在实际产生术语或决策时自动生成。

## 文件结构

本项目使用单上下文模式：

```
/
├── CONTEXT.md          ← 领域术语表
├── docs/adr/           ← 架构决策记录
│   ├── 0001-xxx.md
│   └── 0002-xxx.md
└── src/
```

## 使用术语表的词汇

当输出中涉及领域概念时（issue 标题、重构方案、测试命名等），使用 `CONTEXT.md` 中定义的术语，不要用同义词。

如果需要的概念不在术语表中，说明两种可能：要么是你用了项目不使用的表述（应调整），要么是术语表有空白（记录下来，留给 `/grill-with-docs` 补充）。

## 标记 ADR 冲突

如果输出与已有 ADR 矛盾，明确指出而不是静默覆盖：

> _与 ADR-0007（事件溯源订单）矛盾 — 但值得重新讨论，因为……_
